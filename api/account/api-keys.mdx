## List API Keys

<RequestExample>
```bash GET /api/account/api-keys
curl -X GET "https://panel.example.com/api/account/api-keys?page=1&limit=50" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</RequestExample>

Retrieve a paginated list of API keys for the authenticated user.

### Query Parameters

<ParamField query="page" type="number" default="1">
  The page number to retrieve (minimum: 1).
</ParamField>

<ParamField query="limit" type="number" default="50">
  Number of items per page (minimum: 1, maximum: 100).
</ParamField>

### Response

<ResponseField name="data" type="array">
  Array of API key objects.
  
  <Expandable title="API Key Object">
    <ResponseField name="identifier" type="string" required>
      The unique identifier of the API key.
    </ResponseField>
    
    <ResponseField name="description" type="string | null" required>
      A memo or description for the API key.
    </ResponseField>
    
    <ResponseField name="allowed_ips" type="string[]" required>
      Array of IP addresses allowed to use this key. Empty array means no restrictions.
    </ResponseField>
    
    <ResponseField name="last_used_at" type="string | null" required>
      ISO 8601 timestamp of when the key was last used.
    </ResponseField>
    
    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp of when the key was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination metadata.
  
  <Expandable title="properties">
    <ResponseField name="page" type="number" required>
      Current page number.
    </ResponseField>
    
    <ResponseField name="perPage" type="number" required>
      Number of items per page.
    </ResponseField>
    
    <ResponseField name="total" type="number" required>
      Total number of API keys.
    </ResponseField>
    
    <ResponseField name="totalPages" type="number" required>
      Total number of pages.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>
```json Response
{
  "data": [
    {
      "identifier": "clx1a2b3c4d5e6f7g8h9i0j1",
      "description": "Production API Key",
      "allowed_ips": ["192.168.1.1", "10.0.0.1"],
      "last_used_at": "2024-03-15T10:30:00.000Z",
      "created_at": "2024-01-01T00:00:00.000Z"
    }
  ],
  "pagination": {
    "page": 1,
    "perPage": 50,
    "total": 1,
    "totalPages": 1
  }
}
```
</ResponseExample>

## Create API Key

<RequestExample>
```bash POST /api/account/api-keys
curl -X POST "https://panel.example.com/api/account/api-keys" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "memo": "Development API Key",
    "allowedIps": ["192.168.1.100"],
    "expiresAt": "2025-12-31T23:59:59Z"
  }'
```
</RequestExample>

Create a new API key for the authenticated user.

### Request Body

<ParamField body="memo" type="string" optional>
  A description or memo for the API key (maximum 500 characters).
</ParamField>

<ParamField body="allowedIps" type="string[]" optional>
  Array of IP addresses (IPv4 or IPv6) allowed to use this key. If omitted or empty, the key can be used from any IP address.
</ParamField>

<ParamField body="expiresAt" type="string" optional>
  ISO 8601 datetime string indicating when the key should expire. Must be a future date.
</ParamField>

### Response

<ResponseField name="data" type="object">
  The created API key object.
  
  <Expandable title="properties">
    <ResponseField name="identifier" type="string" required>
      The unique identifier of the API key.
    </ResponseField>
    
    <ResponseField name="description" type="string | null" required>
      The description for the API key.
    </ResponseField>
    
    <ResponseField name="allowed_ips" type="string[]" required>
      Array of allowed IP addresses.
    </ResponseField>
    
    <ResponseField name="last_used_at" type="null" required>
      Will be null for newly created keys.
    </ResponseField>
    
    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp of creation.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  Additional metadata containing the secret token.
  
  <Expandable title="properties">
    <ResponseField name="secret_token" type="string" required>
      The actual API key token. **This is only returned once during creation.**
    </ResponseField>
  </Expandable>
</ResponseField>

<Warning>
  The `secret_token` in the `meta` field is only returned during key creation. Make sure to save it securely - you won't be able to retrieve it again.
</Warning>

<ResponseExample>
```json Response
{
  "data": {
    "identifier": "clx1a2b3c4d5e6f7g8h9i0j1",
    "description": "Development API Key",
    "allowed_ips": ["192.168.1.100"],
    "last_used_at": null,
    "created_at": "2024-03-15T10:30:00.000Z"
  },
  "meta": {
    "secret_token": "xyp_1a2b3c4d5e6f7g8h9i0j1k2l3m4n5o6p7q8r9s0"
  }
}
```
</ResponseExample>

### Error Codes

<ResponseField name="400" type="error">
  Invalid request body, invalid IP address format, or expiration date is not in the future.
</ResponseField>

## Delete API Key

<RequestExample>
```bash DELETE /api/account/api-keys/{identifier}
curl -X DELETE "https://panel.example.com/api/account/api-keys/clx1a2b3c4d5e6f7g8h9i0j1" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</RequestExample>

Delete an API key by its identifier.

### Path Parameters

<ParamField path="identifier" type="string" required>
  The unique identifier of the API key to delete.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Always `true` if the deletion was successful.
</ResponseField>

<ResponseExample>
```json Response
{
  "success": true
}
```
</ResponseExample>

### Error Codes

<ResponseField name="404" type="error">
  API key not found or does not belong to the authenticated user.
</ResponseField>
